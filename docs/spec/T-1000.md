# T-1000 - GitHub Integration Specification

## Objetivo
Adicionar fluxo assistido de publicacao GitHub no ESAA com fallback quando `gh` nao estiver disponivel.

## Escopo
1. `esaa github check`
- Verifica prerequisitos: git repo, branch atual, remoto `origin`, disponibilidade de `gh` e autenticacao (quando aplicavel).

2. `esaa github publish`
- Fluxo assistido para:
  - validar estado local
  - criar repositorio remoto via `gh` (opcional)
  - configurar remote
  - executar push
- Suporte a `--dry-run` para seguranca operacional.

3. Fallback sem `gh`
- Se `gh` indisponivel, retornar plano com comandos `git remote add` + `git push` para uso manual.

## Requisitos
- R1: Comandos devem responder em JSON com status e passos executados/sugeridos.
- R2: `github publish` deve recusar operacao sem git repo inicializado.
- R3: `github publish --dry-run` nao altera estado e retorna plano de comandos.
- R4: Erros devem incluir `next_step`.

## Criterios de Aceitacao
- CA1: `python -m esaa github check` retorna diagnostico completo.
- CA2: `python -m esaa github publish --dry-run` retorna plano assistido.
- CA3: Sem `gh`, o comando retorna fallback acionavel sem falhar de forma opaca.
